SO-ARM101 项目架构总览

无硬件仿真学习 Pipeline — MuJoCo 仿真 + LeRobot + 远程 GPU 训练 — 最后更新 2026-05-10

核心理念

在本地 Windows（无独显）上用 MuJoCo 仿真采集 SO-101 机械臂数据，通过 HuggingFace Hub 同步到远程 RTX 6000D 上训练策略模型，最终实现 sim-to-real 部署。

Fork 自 horndeer/SO-ARM101-LeRobot（TheRobotStudio & HuggingFace），本项目聚焦于无硬件训练 Pipeline。

训练层 Training

本地 CPU (验证)

  • PushT Diffusion Policy — 验证 pipeline
  • SmolVLA 模型加载 (450M params)
  • 数据格式转换 / 可视化

远程 RTX 6000D (训练)

  • ACT (52M params) — 主流 SO-101 策略 Running
  • input: RGB image (480x640) + joint state (6D)
  • output: 100 步 action chunk (6D per step)
  • batch_size=64, num_workers=4, lr=1e-5

已知问题与经验

问题原因解决
GPU 利用率 0%pyav 视频解码是 CPU bound，52M 模型太小无法饱和 RTX6000正常现象，增加 num_workers 提高数据吞吐
groot_n1.py 报 TypeErrorPython 3.12 不允许 non-default field 跟在 default field 后给 4 个 field 加 default=None
batch_size=128 反而慢 10xnum_workers=0 时视频解码成同步瓶颈batch_size=64 + num_workers=4
pip 安装无输出Windows 上 pip.exe 输出被吞用 python -m pip install
davidlinjiahao 模型 404HF 模型可能被删除/改名/设为私有参考其他 SO-101 模型
